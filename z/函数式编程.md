# 函数式编程基础概念文档

函数式编程是一种以“函数”为核心的编程范式，核心是避免副作用、依赖纯函数，且数据倾向于不可变。

## 一、核心定义

函数式编程将计算视为函数的组合，强调以下三个核心：
1. 函数是“一等公民”，可像变量一样被赋值、传递和返回。
2. 避免“副作用”，即函数不修改外部状态（如全局变量、输入参数）。
3. 数据“不可变”，数据创建后不再修改，而是通过函数生成新数据。

## 二、关键特性

1. **纯函数**：输入相同则输出必然相同，且无任何副作用（如打印日志、修改外部变量）。例如 `f(x) = x + 1` 是纯函数，而修改全局变量的函数不是。
2. **函数是一等公民**：支持将函数作为参数传入其他函数（如回调函数），或作为返回值输出。例如 JavaScript 中的 `map(arr, (item) => item * 2)`。
3. **数据不可变**：禁止直接修改已创建的数据，需通过函数生成新数据。例如在 Python 中，列表拼接 `new_list = old_list + [1]` 而非 `old_list.append(1)`。

## 三、常用核心术语

| 术语         | 定义                                                                 |
|--------------|----------------------------------------------------------------------|
| 高阶函数     | 可接收函数作为参数，或返回函数的函数（如 `map`、`filter`）。         |
| 柯里化       | 将多参数函数拆分为一系列单参数函数，逐步接收参数并返回新函数。       |
| 不可变数据结构 | 一旦创建就无法修改的数据类型（如 Scala 的 `List`、Haskell 的 `Data.Map`）。 |
