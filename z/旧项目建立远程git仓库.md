# 旧项目建立远程git仓库 

直播使用了 bigbluebutton 项目（一个开源的直播平台），但是没有添加到自由仓库中进行管理。而是直接拉去后在生产环境进行部署（远程仓库是官方的github仓库）。后面增加功能时都是 另一套代码上修改，然后一个一个文件同步在 生产环境中修改。
现在需要建立一个自有仓库统一管理代码版本，但是 项目源码好像已经被污染了，有 6.9G（多的依赖、副产物等）。

```bash
root@live-uat:/home/boc-live# du -sh bbb-docker-3.0.8-release/
6.9G    bbb-docker-3.0.8-release/
```

有什么办法能把关键源码提取出来，然后建立新的自有仓库嘛

[[git差异覆盖法]]
